# Руководство пользователя: Система управления магазином стройматериалов (BuildStore Backend)

## 1. Введение

BuildStore Backend — это модульная монолитная система для управления процессами оптово-розничного магазина строительных материалов. Система обеспечивает управление товарным каталогом, складскими запасами, процессами закупок и продаж, финансовыми операциями и аудитом.

## 2. Роли и доступ (Role-Based Access Control)

Система построена на строгом разграничении прав доступа:

| Роль | Основные функции |
| :--- | :--- |
| `ROLE_CUSTOMER` | Просмотр каталога, оформление и отслеживание своих заказов. |
| `ROLE_SALES_MANAGER` | Управление заказами клиентов, работа с клиентами. |
| `ROLE_WAREHOUSE_MANAGER`| Инвентаризация, приемка товара, перемещения, отгрузки. |
| `ROLE_PURCHASING_MANAGER`| Закупки у поставщиков, управление входящими накладными. |
| `ROLE_ACCOUNTANT` | Платежи, финансовый леджер, инвойсы, акты. |
| `ROLE_AUDITOR` | Чтение журнала аудита. |
| `ROLE_ADMIN` | Управление пользователями, настройка системы, аудит. |

## 3. Бизнес-процессы

### 3.1. Каталог и цены
*   **Каталог**: Управление продуктами, категориями и единицами измерения. Клиенты видят только активные товары.
*   **Pricing**: Автоматический подбор цен на основе типа клиента (`RETAIL`, `WHOLESALE`, etc.) и актуальных прайс-листов.

### 3.2. Склад и закупки
*   **Закупки**: Создание заказов поставщикам, приемка товаров (Goods Receipt), фиксация расхождений.
*   **Инвентаризация**: Учет остатков (доступных, зарезервированных, поврежденных).
*   **Движение товара**: Каждая складская операция (приемка, отгрузка, списание) фиксируется как неизменяемое `StockMovement`.

### 3.3. Продажи и оплата
*   **Заказы**: Процесс от оформления корзины до отгрузки. Включает резервирование товара на складе.
*   **Оплата**: Поддержка различных методов (Cash, Card, Bank Transfer). Каждая операция требует идемпотентности.

### 3.4. Бухгалтерия и аудит
*   **Инвойсы**: Генерация финансовых документов для заказов.
*   **Audit Log**: Все критические операции (изменение цен, остатков, финансовые проводки) логируются в неизменяемом журнале аудита.

## 4. API и безопасность

*   **Аутентификация**: JWT (JSON Web Token).
*   **Безопасность**:
    *   Принципы Ownership: Клиенты видят только свои данные.
    *   Критические действия (например, ручные корректировки) требуют обоснования (reason).
    *   Все финансовые операции идемпотентны (использование `Idempotency-Key`).

## 5. Обработка ошибок

Система использует единый формат ответов об ошибках, включающий временную метку, статус, код ошибки и детальное сообщение.
Пример:
```json
{
  "timestamp": "2026-06-17T12:00:00Z",
  "status": 409,
  "code": "INSUFFICIENT_STOCK",
  "message": "Not enough stock to reserve...",
  "path": "/api/orders"
}
```
